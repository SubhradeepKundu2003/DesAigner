TD Monthly Newsletter — Pipeline

Upload source documents and run the agent chain.

The chain runs in order:

  1. Agent #1 — Document Ingestion: extracts each file into normalized blocks.
  2. Agent #2 — Content Understanding: the LLM (qwen3.5:4b) identifies & classifies newsletter-worthy items.
  3. Agent #3 — Content Planning: scores each item's impact, selects what makes the issue, and orders the newsletter sections.
  4. Agent #4 — Content Generation: writes each section's articles and the Leadership Message in a consistent corporate tone.
  5. Agent #5 — Fact Validation: checks every article's numbers, dates, and names against the source documents and gates export on high-severity flags.
  6. Agent #6 — Brand Compliance: enforces the brand rulebook (approved terminology, name casing, banned buzzwords) and auto-fixes articles where safe.
  7. Agent #7 — Design Composition: lays out the issue on positioned pages (a template-driven, deterministic Design Model) and renders an HTML preview.
  8. Agent #8 — Image & Graphics: fills article image slots from source-document pictures or approved brand assets, wherever there's room.
  9. Agent #9 — Review: a quality pass over the finished design — deterministic layout checks (overflow, overlaps, margins, contrast) plus an LLM editorial pass on every article, rolled up into a 0–100 quality score.

Note: Agents #2–#6 and #9 call a local Ollama model, so a run can take a little while per document.
